Prompt Engineering for AI Product Managers

A Practical Playbook for Writing, Evaluating, and Iterating on Prompts in Real-World AI Products

1. Foundations

What Prompt Engineering Is (Product-Level Definition)

Prompt engineering is the deliberate design of instructions given to an AI system to produce reliable, controllable, and product-aligned outputs.

For product managers, prompts are not “chat inputs.” They are behavioral specifications for probabilistic systems.

Why It Matters for PMs

  • Determines product quality without changing core model
  • Controls tone, format, compliance, and hallucination risk
  • Reduces engineering overhead
  • Directly impacts user trust

Common Misconceptions

  • “Longer prompts are better.” → Clarity beats verbosity.
  • “The model understands intent.” → It predicts patterns, not goals.
  • “One good prompt works everywhere.” → Context-specific tuning is required.

2. Writing Excellent User Prompts

The CLEAR Framework

  • Context – What situation are we in?
  • Logical Task – What exactly should be done?
  • Expected Output – Format and structure
  • Audience – Who is this for?
  • Restrictions – Constraints and exclusions

Bad → Improved Example #1

Bad Prompt:
Summarize this customer feedback.
Improved Prompt:
Summarize the following customer feedback into 3 bullet points focusing only on product usability issues. Exclude emotional tone. Output in concise bullets.

Product Use Case Example

Use Case: AI-powered survey feedback tool

Generate actionable feedback on this survey question. Identify ambiguity, bias, and clarity issues. Provide revision suggestions in bullet format.

Common Mistakes

  • Missing output format
  • No constraint on length
  • Overly abstract instruction (“Make it better”)
  • No negative instructions (“Do not include…”) when necessary

Bad → Improved Example #2

Improve this email.
Rewrite this email to be concise, professional, and under 120 words. Preserve all deadlines and commitments. Do not change factual content.

3. Writing Strong System Prompts

Core Components

  • Role Framing: Define expertise and perspective
  • Constraints: Word limits, tone, exclusions
  • Output Control: Structured formats
  • Tone: Formal, friendly, neutral
  • Guardrails: Refuse unsafe or irrelevant requests

Example: AI Support Assistant

You are a SaaS customer support assistant. Provide accurate answers using only verified product documentation. If uncertain, say “I don’t have enough information.” Always respond in concise paragraphs under 150 words.

Example: AI Hiring Assistant

You are a hiring assistant evaluating resumes. Score candidates on experience relevance (0–5), skills match (0–5), and clarity (0–5). Provide reasoning for each score in bullet format.

4. Prompt Evaluation Framework

The PACE Rubric

  • Precision – Is the task unambiguous?
  • Alignment – Does it match product goals?
  • Constraints – Are limits defined?
  • Edge Cases – Are failure modes considered?

Scoring Checklist (1–5 Each)

  • Clarity
  • Specificity
  • Constraints
  • Edge Case Handling
  • Alignment with Product Intent

Evaluating a Weak Prompt

Write onboarding instructions for new users.

Evaluation:

  • Clarity: 2/5 (Which product?)
  • Specificity: 1/5
  • Constraints: 0/5
  • Edge Cases: 0/5
  • Alignment: 2/5
Improved Version:
Write onboarding instructions for first-time users of a B2B analytics dashboard. Keep under 300 words. Use numbered steps. Assume non-technical users. Focus on account setup and first report generation.

5. Iteration and Debugging Prompts

Systematic Testing

  • Test with varied inputs
  • Test edge cases deliberately
  • Compare outputs side-by-side
  • Log failure patterns

Failure Pattern Diagnosis

  • Too verbose → Add word limits
  • Hallucination → Add sourcing constraints
  • Inconsistent format → Enforce structure explicitly

When to Adjust What

  • Prompt issue → Rewrite instruction
  • Knowledge gap → Add retrieval (RAG)
  • Reasoning limitation → Upgrade model

Practical Iteration Workflow

  1. Define success criteria
  2. Draft baseline prompt
  3. Test against 5–10 realistic inputs
  4. Score using PACE rubric
  5. Refine constraints and structure
  6. Repeat until consistent ≥4/5 across dimensions
